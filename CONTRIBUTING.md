# Contributing to FrankX Workshops

Thank you for your interest in improving our workshops! This guide will help you contribute effectively.

## Quick Start

1. Fork this repository
2. Create a branch: `git checkout -b feat/my-improvement`
3. Make your changes
4. Submit a pull request

## Quality Standards

Every contribution must meet these standards:

### Content Quality
- Clear, concise writing
- Technical terms explained on first use
- Practical, real-world examples
- No spelling or grammar errors

### Code Examples
- All code must be tested and working
- Use modern best practices (ES modules, TypeScript)
- Include comments for complex sections
- Show error handling where appropriate

### Workshop Structure
- Maintain 40%+ hands-on content ratio
- Include clear learning objectives
- All links must be working
- Follow existing formatting conventions

## Types of Contributions

### Bug Fixes
Found something that doesn't work? Submit a fix!

1. Check if an issue already exists
2. If not, open an issue describing the bug
3. Submit a PR referencing the issue

### Content Improvements
- Clearer explanations
- Better examples
- Updated information
- Fixed typos

### New Content
- New modules for existing workshops
- New labs or exercises
- New code examples

### New Workshops
Want to create a new workshop? Open an issue to discuss first. Include:
- Workshop topic and scope
- Target audience
- Proposed module structure
- Why this adds value

## Pull Request Process

1. **Title**: Use conventional commits format
   - `feat: Add MCP authentication module`
   - `fix: Correct PostgreSQL connection example`
   - `docs: Improve CLAUDE.md explanations`

2. **Description**: Explain what and why

3. **Checklist**: Complete the PR template checklist

4. **Review**: Address any feedback

## Code of Conduct

- Be respectful and constructive
- Focus on the work, not the person
- Help others learn and improve

## Recognition

Contributors are recognized in:
- README acknowledgments
- Workshop credits
- Annual contributor highlights

## Questions?

- Open a GitHub Discussion
- Join our Discord community
- Check existing issues
